Click-Compress

Smart compression platform for every file type

A web platform that shrinks video, PDF, images, and documents using the right algorithm for each format — not a one-size-fits-all zip tool.

Project overview · 6 slides

The problem

  • Files are too large to share, upload, or store easily.
  • Generic compressors treat all files the same and save little on media.
  • Cloud tools raise privacy concerns for personal and work documents.
  • Users want clear savings (e.g. ~40%) without broken or unplayable output.

Our solution

Click-Compress inspects each upload and routes it to a specialized pipeline:

  • Target 40% — re-encode when needed to hit size goals.
  • High impact local — ffmpeg and Ghostscript on your machine.
  • Strict lossless — byte-perfect restore for archives and text.

Browser-first where possible; native tools for video and PDF.

What we compress

Type Method Typical savings
Video (MP4, MOV…) H.264 / H.265 + AAC 40–80%
PDF Ghostscript profiles 30–50%
Images WebP / JPEG sweep 35–60%
Office & text Brotli / Gzip 20–90%

How it works

  1. Upload — user picks file, goal, and profile.
  2. Analyze — dispatcher detects type (video, PDF, image, text…).
  3. Compress — best engine runs (browser or native).
  4. Download — optimized file with savings stats and method shown.

Video outputs are validated for QuickTime compatibility before download.

Technology & summary

Stack: Next.js, TypeScript, brotli-wasm, fflate, ffmpeg, Ghostscript

  • Multi-page platform: Home, Capabilities, Compress, How it works, About
  • Local processing — no external compression API required
  • Honest reporting when files are already highly compressed

Goal: Make large files smaller, playable, and shareable — simply.
